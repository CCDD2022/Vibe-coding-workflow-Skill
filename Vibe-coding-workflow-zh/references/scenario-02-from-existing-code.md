# 场景 2：基于已有代码写代码

## 输入模板（必填）

- 本次新增需求（必须）
- 不可改动范围（可选）
- 验收标准（必须）
- 改哪里（可留空，由 AI 自行定位）

## 执行流程（固定 5 步）

1. **先读 `structure.md` 并核对实际代码结构**

- 对不明确部分，直接打开对应文件阅读。
- 若 `structure.md` 与真实项目结构差异较大：
  - 直接阅读完整项目结构；
  - 先更新 `structure.md`，再进入下一步。

1. **读取 `goal.md`，结合用户需求生成详细 `plan.md`**

- 将需求拆解为可执行子任务、影响模块、验收条件。
- 若用户未提供“改哪里”，由 AI 先给出改动点候选清单，再确认。

1. **与用户沟通并迭代 `plan.md`，直到用户同意**

- 未获用户同意，不进入编码阶段。

1. **按 `plan.md` 定位相关代码并做详细阅读**

- 逐个阅读目标模块、调用链、关键数据结构与边界条件。

1. **执行编码实现**

- 严格按已同意的 `plan.md` 开发。
- 编码后按需回填更新文档（`plan.md` / `structure.md` / 其他核心文档）。

## 输出物清单

- 更新后的 `structure.md`（如有差异）
- 已确认的 `plan.md`
- 变更清单（模块/文件/影响范围）
- 验证记录（新增需求 + 回归）
- 文档回填记录

## 完成判定

- [ ] `structure.md` 与实际结构一致（必要时已更新）
- [ ] `plan.md` 已详细生成并经用户同意
- [ ] 新需求通过
- [ ] 旧功能未破坏
- [ ] 变更可追溯
